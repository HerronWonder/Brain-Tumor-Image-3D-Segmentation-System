# 论文目标对齐清单

## 说明
本清单按“系统工程目标”与“实验目标”两条线拆分，状态分为：
- DONE: 已完成并已落代码
- PARTIAL: 已有基础，但未达论文交付标准
- TODO: 尚未完成

## A. 系统工程目标

1. Spring Boot + FastAPI 职责分层: DONE
- Spring Boot: 对外 API、上传下载、任务编排
- FastAPI: 模型推理内核
- 前端仅调用网关

2. 多模态输入严格校验: DONE
- 必须 4 模态且命名可识别（t1/t1ce/t2/flair）

3. 临床体积指标计算可信化: DONE
- 使用 NIfTI spacing 计算体素体积

4. 多模型在线切换（unet/mamba）: DONE
- 网关与推理服务都支持模型字段

5. 链路可追踪（request-id）: DONE
- 网关生成/透传 X-Request-ID，推理服务回写

6. 内部服务鉴权: DONE
- 支持 X-Internal-Token（可选启用）

## B. 论文实验与交付目标

1. 指标体系完整对齐论文（Dice/HD95/各分区等）: PARTIAL
- 当前在线接口主要返回体积指标
- 离线测试脚本已有 Dice 基础，但未统一到论文汇报口径

2. 可复现实验配置（固定随机种子、配置快照、版本锁定）: PARTIAL
- 有训练脚本与 wandb 记录
- 仍存在部分路径/参数硬编码

3. 自动化测试覆盖（后端接口、前端组件、端到端）: TODO
- 当前以手工验证为主，缺少稳定 CI 级测试

4. 部署与运行手册（开发/生产分层）: PARTIAL
- 已有 start.sh 与架构文档
- 生产部署脚本与健康探针规范仍需补充

5. 结果可视化与论文图表自动化输出: PARTIAL
- 已有相关脚本
- 仍需统一输入输出协议与批处理入口
